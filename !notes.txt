# Projekt PROJOPW_PKGE - Struktura folderów i ich zawartość

## Główne foldery projektu:

1. **CSS/**
    - Zawiera style CSS podzielone na moduły.
    - Podfoldery:
      - **base/** - Podstawowe style, np. reset CSS, typografia.
      - **components/** - Style dla wspólnych elementów używanych na wielu stronach, takich jak przyciski, formularze, karty, itp.
      - **layout/** - Style układu strony, np. nagłówki, stopki, siatki.
      - **pages/** - Style specyficzne dla poszczególnych stron (np. index.html, poziom1.html).
      - **theme/** - Style tematyczne, np. tryb ciemny (dark mode).
    - Plik:
      - **main.css** - Główny plik CSS, który importuje wszystkie inne style.

2. **IMG/**
    - Zawiera pliki graficzne używane w projekcie, takie jak obrazy, ikony.
    - Podfoldery:
      - **backgrounds/** - Obrazy tła używane w projekcie.
        - Przykład: `header-bg.jpg` - obraz tła dla nagłówka.
      - **content/** - Obrazy specyficzne dla poszczególnych stron.
        - **index/** - Obrazy używane na stronie głównej.
          - Przykład: `hero-image.jpg` - obraz banera na stronie głównej.
        - **poziom1/** - Obrazy używane na stronie poziom1.html.
          - Przykład: `task-image.jpg` - obraz zadania na poziomie 1.
        - **poziom2/** - Obrazy używane na stronie poziom2.html.
        - **poziom3/** - Obrazy używane na stronie poziom3.html.
      - **icons/** - Ikony używane w interfejsie.
        - Przykład: `home-icon.svg` - ikona dla przycisku "Home".

3. **JS/**
    - Zawiera pliki JavaScript odpowiedzialne za logikę aplikacji.
    - Podfoldery:
      - **modules/** - Moduły wielokrotnego użytku, np. funkcje, klasy.
        - Przykład: `formHandler.js` - obsługa formularzy.
      - **pages/** - Skrypty specyficzne dla poszczególnych stron.
        - Przykład: `index.js` - skrypt specyficzny dla strony głównej.
      - **utils/** - Funkcje pomocnicze, np. formatowanie dat, walidacja.
        - Przykład: `dateFormatter.js` - funkcja do formatowania dat.
    - Plik:
      - **main.js** - Główny plik JS, który importuje inne moduły i inicjalizuje aplikację.

4. **index.html**
    - Strona główna projektu.

5. **poziom1.html**, **poziom2.html**, **poziom3.html**
    - Strony reprezentujące różne poziomy aplikacji.

6. **!notes.txt**
    - Plik z notatkami dotyczącymi struktury projektu i jego zawartości.

## Ładowanie stylów:
- **main.css**:
  - Zawiera wspólne style dla całego projektu, takie jak nagłówki, stopki, przyciski.
  - Jest ładowany na każdej stronie jako pierwszy.
- **Specyficzne style dla stron**:
  - Każda strona (np. poziom1.html) ma swój własny plik CSS w folderze `pages/` (np. `poziom1.css`).
  - Plik specyficzny dla strony jest ładowany po `main.css`, aby nadpisać wspólne style tam, gdzie to konieczne.

### Przykład ładowania stylów w HTML:
```html
<link rel="stylesheet" href="CSS/main.css"> <!-- Wspólne style -->
<link rel="stylesheet" href="CSS/pages/poziom1.css"> <!-- Style specyficzne dla poziom1 -->
